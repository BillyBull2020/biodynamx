# Web 4.0 Standard — BioDynamX Engineering Group

> Every website, app, landing page, agent, and feature we build from this point forward is built on Web 4.0. No exceptions.

## What Web 4.0 Means at BioDynamX

Web 4.0 = **Ambient Intelligence** — the web that thinks, adapts, and acts on behalf of the user without being explicitly asked. It is the evolution from:

- Web 1.0 → Static pages (read only)
- Web 2.0 → Social/interactive (read + write)
- Web 3.0 → Decentralized/semantic (own your data)
- **Web 4.0 → Ambient + agentic (the web acts FOR you)**

## Required on Every Build

### 1. Ambient Intelligence Layer

Every page must sense and respond to user context without explicit commands:

- Scroll position awareness (content adapts as user scrolls)
- Idle detection (triggers engagement after inactivity)
- Exit intent (activates offer/agent before user leaves)
- Time-on-page personalization (message shifts based on engagement depth)
- Geo-awareness (content reflects user's city/industry)

### 2. Agentic Layer

Every product must include or connect to an AI agent that can:

- Initiate conversations proactively (not wait to be clicked)
- Use voice — not just text — as a first-class interface
- Learn from the interaction and adapt in real time
- Execute actions autonomously (book, send, close, follow up)

### 3. AI/LLM Discoverability (GEO + AEO)

Every page must be optimized for AI search engines, not just Google:

- `llms.txt` — machine-readable site summary at root
- `llms-full.txt` — complete structured content for LLMs
- `robots.txt` — explicitly allow GPTBot, Google-Extended, PerplexityBot, ClaudeBot, Anthropic
- `.well-known/ai-agent-policy.json` — agentic permissions manifest
- `.well-known/did.json` — decentralized identity (DID Web)
- JSON-LD schemas — Organization, SoftwareApplication, FAQPage, HowTo, WebSite, Person
- All schemas must include disambiguation if brand name is ambiguous

### 4. Decentralized Identity

Every BioDynamX property must publish:

- `did:web:biodynamx.com` — verifiable identity
- Signed agentic action permissions
- Transparent data usage declarations

### 5. Conversational UI as First-Class Interface

Every page should have a voice/chat interface as the PRIMARY call to action — not a button that leads to a form. The agent IS the UX.

### 6. Symbiotic Web Principles

Every build must:

- Adapt to the user (not force the user to adapt to it)
- Be transparent about what AI is doing and why
- Protect user privacy while delivering personalization
- Feel alive — micro-animations, ambient visuals, reactive elements

## Web 4.0 Messaging Standards

### Hero Copy (English)

Always mention Web 4.0 as a competitive differentiator. Position it as the **unfair advantage**:
> "...they run on Web 4.0: the ambient, intelligent web that thinks, adapts, and acts before your competitor even wakes up."

### Sub-badges

Both badges must appear together in the hero:

- 🧠 **Powered by Neuroscience & Neuromarketing** (purple)
- 🌐 **WEB 4.0 NATIVE** (green)

### AI/LLM Copy

In `llms.txt`, `llms-full.txt`, and all AI-readable content:
> "BioDynamX is a Web 4.0 AI platform — built on ambient intelligence, agentic AI, and the Symbiotic Web."

## Checklist for Every New Build

Before deploying any new page, feature, or site:

- [ ] Ambient intelligence layer present (scroll/idle/exit/geo)
- [ ] Voice or conversational UI available
- [ ] `llms.txt` updated to include new page/feature
- [ ] JSON-LD schema added for new page
- [ ] Web 4.0 badge or mention present in hero/above fold
- [ ] robots.txt allows major AI crawlers
- [ ] Agent can act autonomously (not just respond)
- [ ] All ROI claims consistent (5x guarantee — or your money back)
- [ ] No fabricated statistics — all numbers are verifiable or labeled as examples
- [ ] Deployed to Firebase with `firebase deploy --only hosting`
