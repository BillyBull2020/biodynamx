# Neuro-Conversion Audit Workflow

This workflow is **MANDATORY** and must be executed on every website, app, or landing page we build or significantly modify.

## When to Trigger

- After building a new website or landing page
- After significant UI/UX changes
- When reviewing a client's site via the BioDynamX audit tool
- Before final delivery of any web project

## Steps

1. **Read the SKILL.md** — Load the full neuro-conversion audit framework:

   ```
   View file: .agent/skills/neuro-conversion-audit/SKILL.md
   ```

2. **Capture the Current State** — Take a screenshot or review the DOM of the page being audited.

3. **Run the 5-Framework Audit** — Evaluate the page against:
   - Framework 1: Cognitive Load & Friction (Ability)
   - Framework 2: Cognitive Biases & Dopamine Economy
   - Framework 3: Social Proof & Trust Elements
   - Framework 4: Fogg Behavior Model (B=MAP)
   - Framework 5: Irresistible Offer Positioning

4. **Output the Audit Report** — Format results as:
   - ✅ Strengths (what's working)
   - 🔴 Friction Points (what's broken)
   - 📋 Actionable Recommendations (exact changes needed)

5. **Implement Critical Fixes** — Apply the highest-impact recommendations immediately:
   - CTA optimization
   - Price anchoring
   - Loss-framed copy
   - Trust element placement
   - Friction reduction

6. **Verify Changes** — Re-check the page to confirm improvements were applied correctly.
